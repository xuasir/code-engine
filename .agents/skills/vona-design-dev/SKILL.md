# vona-design-dev

本技能用于“功能设计”阶段，遵循渐进式剖析与类型优先原则，结合仓库设计提纲与业内最佳实践，稳定输出可审阅的设计产物与验证清单。

## 适用场景
- 进行功能/模块/API 的设计、架构调整、模块拆分与整合
- 开始设计评审或需要形成可落地的设计说明与任务清单
- 需要在既有代码结构中定位承载点并规划改造路线

## 输入
- 需求描述：做什么、为何做、成功判据
- 约束条件：性能目标、可维护性、兼容要求、风险边界
- 使用方式与代码风格偏好：期望的调用体验或 API 风格文档
- 现有代码线索：相关包/模块位置、已存在的能力或限制

## 输出
- 需求要点清单与范围界定
- 关键功能点与职责划分
- 接口层设计（类型优先）：Module Kit / Core Module Kit 的对外接口面
- 关键逻辑实现路径与数据流示意（文字化步骤与边界）
- 变更项列表：新增/调整的 Kit、Core Kit、配置 Schema
- 验证清单：覆盖功能正确性、类型与结构一致性、边界与性能、测试与集成
- 后续行动项：落地任务与里程碑

## 流程（分阶段）
### 阶段 A｜明确需求
- 提炼目标、约束、成功指标；识别范围与不做事项
- 发现相关包与现有能力，标注承载点与冲突/空白区

### 阶段 B｜反向工程设计
- 从需求抽取关键功能点与协作关系
- 获取/确认期望的使用方式与 API 风格（若未提供则给出建议范式）
- 基于类型优先进行接口层设计：Module Kit / Core Module Kit
- 规划关键逻辑路径：输入/输出、状态与错误、扩展点与限流/缓存策略

### 阶段 C｜总体确认
- 汇总新增模块、Core 能力与配置 Schema 变更
- 明确职责边界、扩展点、兼容与迁移策略、风险与监控
- 发布设计产物与验证清单，形成后续行动项

## 决策点
- 能力归属：Kit vs Core；运行时参数 vs 配置 Schema
- 扩展机制：是否引入插件/钩子；稳定接口与版本策略
- 性能与资源：热路径、缓存/并发控制、错误降级
- 依赖最小化：避免循环依赖与跨层耦合

## 验证清单
- 设计产物齐备：要点清单、接口层设计、逻辑路径、变更项、行动项
- 类型与边界明确：输入/输出类型、错误与状态流、扩展点
- 包结构与职责清晰：模块边界、对外 API、一致的命名与导出
- 配置 Schema 可表达且可演化：默认值、校验与向后兼容
- 测试与集成可行：单测/集成测试建议、模拟策略、验证场景
- 风险与性能评估：热路径与上限、降级与观测指标

## 使用说明
- 当进入“功能设计/方案/API 设计/评审”场景时立即调用本技能
- 技能将自动检索仓库以定位相关模块，并在必要时提出 1–4 个关键问题补全信息
- 产出以结构化要点与清单为主，避免不必要的文件变更与提交

## 参考与约束
- 遵循业内最佳实践（可组合、可触发、输入/输出明确、原子且稳健）
- 不引入多余依赖；不提交代码；尽量沿用既有构建/类型/测试流程
